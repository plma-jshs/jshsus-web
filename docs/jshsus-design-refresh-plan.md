# JSHSUS Design Refresh Plan

작성일: 2026-06-22

## 1. 전제

첨부 이미지 파일은 현재 세션에서 읽을 수 없었다. 따라서 이 계획은 다음 두 가지 근거를 기준으로 작성한다.

- 기존 공개 메인페이지 `https://jshsus.kr/`에서 확인한 정보 구조
- 현재 리뉴얼 코드의 `apps/web/src/styles.css`, `apps/web/src/router.tsx`, `apps/admin/src/styles.css` 구조

이미지가 다시 제공되면 색감, 로고 위치, 카드 밀도, 실제 첫 화면 구도만 추가 보정한다.

## 2. 현재 리뉴얼 화면의 문제

- 학생 포털이 학교 포털보다는 운영 대시보드처럼 보인다.
- 좌측 사이드바와 흰색 패널 위주의 반복 구조라 기존 과구리의 “서비스 허브” 느낌이 약하다.
- 첫 화면에서 `과구리`, 전남과학고, 학교생활 서비스라는 정체성이 작게만 보인다.
- 공지, 게시판, 청원, 분실물, 탐활서가 모두 같은 카드 톤이라 중요도 차이가 없다.
- 색상은 민트/회색 위주로 단조롭고, 기존 사이트의 다기능 포털 느낌을 이어받지 못한다.
- 이미지나 실제 학교 맥락을 보여주는 시각 자산이 없어 모던함보다 임시 관리자 화면처럼 느껴진다.

## 3. 기존 과구리에서 유지할 감각

기존 메인페이지는 다음 구조가 강하다.

- 상단 브랜드: `과구리`
- 주요 링크: 소개, 학생부 전산시스템
- 전체 서비스 목록: 공지사항, 탐활서, 자유게시판, 청원·제안, 상벌점/휴대폰, JBS, 분실물, 기상곡 신청, 세특 바이트 계산기 등
- 메인 콘텐츠: 공지사항, 환영 메시지, 로그인, 학사일정, 자유게시판, 청원·제안
- 하단: IT부/개인정보처리방침/서비스 안내

리뉴얼에서는 이 정보 구조를 버리지 않고, “낡은 포털”을 “모던한 학교 생활 허브”로 재해석한다.

## 4. 디자인 목표

1. 첫 화면에서 브랜드와 학교생활 포털 성격이 즉시 보여야 한다.
2. 기존 과구리의 서비스 밀도는 유지하되, 카드 나열이 아니라 우선순위가 있는 포털 홈으로 재구성한다.
3. 학생 포털과 관리자 포털의 성격을 분리한다.
4. 색상은 단일 민트톤에서 벗어나, 전남과학고/과학/학교생활에 맞는 다색 체계로 확장한다.
5. 실제 학교 이미지 또는 학교 건물/과학실/생활관을 연상시키는 시각 자산을 첫 화면에 사용한다.

## 5. 학생 포털 개편안

### 5.1 레이아웃

현재 좌측 고정 사이드바 중심 구조를 학생 포털에서는 약화한다.

- Desktop
  - 상단: 브랜드 바 + 주요 메뉴 + 로그인/내 상태
  - 첫 화면: 큰 학교 포털 히어로 + 오늘의 상태 패널 + 빠른 서비스
  - 본문: 공지/학사일정/게시판/청원/분실물 2열 그리드
- Mobile
  - 상단: 브랜드, 알림, 전체서비스 버튼
  - 하단: 홈/공지/탐활서/청원/내 상태 탭
  - 전체서비스는 아이콘 그리드 drawer 또는 별도 화면

### 5.2 첫 화면

첫 viewport는 다음 구성으로 바꾼다.

- 왼쪽/중앙: 학교 이미지 기반 히어로
  - H1: `과구리`
  - 보조 문구: `전남과학고 학생 생활 포털`
  - CTA: `탐활서 신청`, `내 생활 상태`
- 오른쪽: 로그인/내 상태 패널
  - 로그인 전: 로그인 버튼, 주요 서비스 안내
  - 로그인 후: 상벌점, 기숙사, 보관함, 최근 탐활서 상태
- 하단: 서비스 런처
  - 공지사항, 탐활서, 자유게시판, 청원·제안, 상벌점/휴대폰, 분실물, 기상곡 신청, 세특 바이트 계산기

### 5.3 색상

단일 민트 중심에서 다음 체계로 확장한다.

- Primary: deep teal `#167C72`
- Secondary: science blue `#315AA8`
- Accent: warm yellow `#F2C14E`
- Positive: green `#2E9D63`
- Warning: amber `#C77700`
- Danger: red `#C24132`
- Surface: white, cool gray, very light blue

민트는 브랜드 축으로 유지하되, 공지/탐활서/청원/분실물/생활상태마다 보조 색을 다르게 사용한다.

### 5.4 컴포넌트

- `PortalHeader`: 기존 과구리 감각을 살린 상단 브랜드/서비스 메뉴
- `HeroStatus`: 학교 이미지 + 환영 문구 + 내 상태 요약
- `ServiceLauncher`: 기존 전체서비스 목록을 아이콘 타일로 재구성
- `NoticeRail`: 고정 공지와 최신 공지를 분리
- `CalendarPanel`: 학사일정 영역 복원
- `CommunityPreview`: 자유게시판과 청원·제안을 묶은 참여 영역
- `LostItemPreview`: 최근 분실/습득 항목

### 5.5 시각 자산

- 1순위: 실제 전남과학고/교내 사진 사용
- 2순위: 학교 건물, 과학실, 기숙사 생활을 담은 생성 이미지
- 3순위: 단순 SVG/그라데이션은 사용하지 않는다.

히어로는 이미지가 배경이고 텍스트는 카드 안이 아니라 이미지 위에 직접 배치한다. 단, 가독성을 위해 어두운 overlay를 얕게 둔다.

## 6. 관리자 포털 개편안

관리자 포털은 학생 포털보다 더 조용하고 밀도 있게 유지한다. 다만 현재보다 도메인 구분을 강화한다.

- 좌측 메뉴는 유지하되, 상벌점/보관함/기숙사/탐활서/청원/콘텐츠/IAM을 그룹화한다.
- 상단에 `오늘 처리할 일` 큐를 고정한다.
- 도메인별 색상 라벨을 부여한다.
  - 상벌점: green/red
  - 보관함: blue
  - 기숙사: violet이 아닌 muted indigo
  - 탐활서: teal
  - 청원/콘텐츠: amber/blue
- 테이블은 zebra row, sticky header, density toggle을 추가한다.
- 위험 작업은 destructive color와 확인 modal을 사용한다.

## 7. 구현 순서

1. 디자인 토큰 정리
   - `apps/web/src/styles.css`의 색상, spacing, card, typography 변수를 재정의한다.
   - 학생/관리자 공통 토큰과 학생 전용 토큰을 분리한다.

2. 학생 포털 홈 개편
   - 사이드바 중심 홈을 상단 브랜드 바 + 히어로 + 서비스 런처 구조로 변경한다.
   - 기존 공지/게시판/청원/분실물 API는 그대로 사용한다.

3. 서비스 런처 구현
   - 기존 사이트의 전체서비스 목록을 유지하면서 새 기능과 연결한다.
   - 외부/미구현 서비스는 “외부 링크” 또는 “준비 중” 상태를 명확히 표시한다.

4. 콘텐츠 영역 개편
   - 공지사항: 고정 공지/최신 공지 분리
   - 자유게시판: 최근 글 + 댓글 수
   - 청원: 참여 progress
   - 분실물: 습득/분실 badge

5. 관리자 포털 시각 보정
   - 현재 기능은 유지하고, 그룹 메뉴와 테이블 밀도/상태 색상만 개선한다.

6. 반응형 QA
   - 모바일 360px, 태블릿 768px, 데스크톱 1440px 기준으로 직접 화면 확인한다.
   - Playwright 자동화는 이번 범위에서 제외한다.

## 8. 완료 기준

- 첫 화면에서 `과구리`와 `전남과학고 학생 생활 포털`이 가장 큰 시각 신호로 보인다.
- 기존 사이트의 전체서비스 목록 감각이 서비스 런처로 재현된다.
- 공지/학사일정/자유게시판/청원/분실물이 첫 화면에서 한 번에 스캔된다.
- 학생 포털은 학교 서비스 허브처럼 보이고, 관리자 포털은 운영 콘솔처럼 보인다.
- 모바일에서 메뉴와 카드 텍스트가 겹치지 않는다.
